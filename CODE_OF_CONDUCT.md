# Code of Conduct

This project expects focused, respectful, and security-conscious collaboration.

## Expected Behavior

- Keep discussions technical, specific, and actionable.
- Respect maintainers' decisions about scope, validation, and deployment safety.
- Avoid posting secrets, credentials, private target code, or local runtime state.
- Assume contributors may be working with different operating systems, shells,
  and Codex runtime setups.

## Unacceptable Behavior

- Harassment, personal attacks, or discriminatory language.
- Publishing private information without permission.
- Deliberately bypassing validation, deployment boundaries, or source/runtime
  separation rules.
- Submitting issues or pull requests that knowingly include secrets or protected
  local files.

## Enforcement

Maintainers may edit, close, or remove issues, pull requests, comments, or other
contributions that violate these expectations. Serious or repeated violations
may result in blocked participation.
