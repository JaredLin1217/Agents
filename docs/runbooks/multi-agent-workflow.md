# Multi-Agent Workflow

Use only after an explicit hire/spawn/delegate/parallel-agent request or clear equivalent.

1. Read `AGENTS.md`; choose `session-managed`, `manual-detached`, or `cloud-or-automation`.
2. Use `explorer` for read-only work; use `worker` only with exclusive normalized write scope.
3. Start from `docs/agent-assignment.template.md`; include required fields from `docs/agents/schemas.yaml`.
4. Create/update the temp roster when employees may outlive context, run concurrently, or edit files. Report XR/XW.
5. Final report is the completion notification. Poll before overlapping edits or closeout.
6. Close by runtime id first; history cleanup is separate and needs explicit authorization.
7. Controller reviews reports and diffs before integration.

References: `docs/agents/workflows.yaml`, `docs/agents/schemas.yaml`, `docs/agents/verify.yaml`.
